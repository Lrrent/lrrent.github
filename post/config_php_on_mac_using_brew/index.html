上面这个地方，在我整个写完文章之后，觉得这个地方应该是有点问题的，按理说不应该在brew update加sudo前缀来使用，这样就违反了Homebrew的使用原则，因为Homebrew不鼓励在一般条件下强行使用sudo，因此，还是应该执行 sudo chown $(whoami) /path 这样的方式，让目录或者目录的权限，归属到当前用户下。

大概看了下上面的输出信息，有几个需要注意的地方：

  1. 可以通过添加系统路径，将 PATH="/usr/local/bin:$PATH"加到系统路径中去
  2. 如果你希望使用PHP命令命令工具，需要更新系统PATH变量，执行这个命令 export PATH="$(brew --prefix homebrew/php/php56)/bin:$PATH"
  3. 如何需要PHP-FPM随服务器启动，则需要执行下面3条命令：
    1.mkdir -p ~/Library/LaunchAgents #参数p是根据需要创建中间目录的意思，但是具体啥意思，还不是很明白。
    1. cp /usr/local/opt/php56/homebrew.mxcl.php56.plist ~/Library/LaunchAgents/ #这里也可以使用ln -
    2. launchctl load -w ~/Library/LaunchAgents/homebrew.mxcl.php56.plist
    3. 控制命令脚本位于 /usr/local/opt/php56/sbin/php56-fpm

错误 Nginx: ‘[emerg] mkdir() “/usr/local/var/run/nginx/client_body_temp”‘

可能的解决办法：升级到至少Yosemite以上的MAC版本，然后重新使用Homebrew强制安装nginx

错误 git: ‘Could not resolve host: git.frd.mn’

可能的解决办法：这可能是我的服务器的问题，请与我联系。

错误 curl: ‘Failed to connect to localhost port 80: Connection refused’

可能的解决办法：这是一个涉及 IPv6 的问题，问题出在你的Mac电脑的 /etc/hosts 文件出。如果想修复这个错误，在hosts文件中，找到 “fe80::1%lo0 localhost” 这一行，并且把他注释掉，或者使用下面的命令一键搞定：

错误 brew: ‘configure: error: Can not find OpenSSL’s ‘

可能的解决办法：确认Xcode已经安装，并且保证Xcode CLI工具安装并且保持最新版本。
