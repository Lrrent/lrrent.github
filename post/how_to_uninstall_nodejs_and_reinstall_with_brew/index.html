mac下如何完整卸载node.js

因为之前使用brew doctor发现了node安装在了/usr/local/include目录下，这样brew就会觉得有文件没有应该放对位置，因此我就要先卸载一次再使用brew来安装它，而不是简单使用node.js的安装包安装。

写了个脚本，也不算脚本吧。

好了

使用brew安装node.js和npm

安装brew，已经安装的可以略过

ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

升级brew
brew update
As a safe measure, run brew doctor to make sure your system is ready to brew. Follow any recommendations from brew doctor.

为了安全升级，先运行检查
brew doctor
添加homebrew的路径到系统PAHT中，已经添加的可以略过，查看使用echo $PATH

export PATH=”/usr/local/bin:$PATH”

安装Node
brew install node

然后，通过node -v和npm -v检查是否安装正确。

如果出错，说command not found，说明有问题。

通过brew doctor检查下

然后会看到node没有被link，使用brew link node，遇到错误

结果

按照建议

  1. rm ‘/usr/local/share/doc/node/gdbinit’
  2. brew link –overwrite node

再次node -v和npm -v

搞定。

#mac #node.js #homebrew #brew #uninstall
